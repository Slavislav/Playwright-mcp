1. Introduction

1.1 Purpose

This Test Plan defines the comprehensive testing strategy for Loox, a Shopify application designed for customer review collection and management. The plan outlines systematic testing approaches to ensure the application meets functional requirements, performance standards, and security compliance within the ISTQB Testing Cup competition framework.

1.2 System Under Test Overview

Loox is a Shopify app that enables merchants to collect, manage, and display customer reviews with photos and videos. The system operates as an embedded iframe-based application within the Shopify ecosystem, providing review collection, management, display widgets, email automation, and discount management capabilities.

Key System Characteristics Observed:
  • Setup Status: Complete (Core script and widgets configured)
  • Current Data State: 0 review requests sent, 0 reviews collected, 0 photo/video reviews
  • Performance Metrics: $0 reviews-generated revenue, $3.4K referrals-generated revenue
  • Main Modules: Reviews (active), Referrals (disabled), Upsell (disabled), Settings (active)

1.3 Test Objectives

The primary objectives of this testing effort are:

  1. Verify core review collection and management functionality
  2. Validate email automation and customization features
  3. Test discount management and code generation
  4. Ensure proper Shopify integration and data synchronization
  5. Validate security and compliance features
  6. Verify performance and scalability requirements

3. Test Approach

3.1 Test Levels

The testing approach encompasses multiple test levels to ensure comprehensive coverage:

3.1.1 Functional Testing

  • Integration Testing: Shopify API integration and data flow
  • System Testing: End-to-end workflow validation
  • User Acceptance Testing: Business process validation
Unit Testing Note: Unit testing is not applicable for this Shopify app as it operates as a black-box iframe-based application. The app's internal components are not accessible for unit testing, and testing focuses on integration and system-level functionality.

3.1.2 Non-Functional Testing

  • Performance Testing: Load testing using K6 + Performance Analyzer
  • Security Testing: Vulnerability assessment using OWASP ZAP
  • Accessibility Testing: WCAG compliance using Google Lighthouse
  • Usability Testing: User experience and interface validation
