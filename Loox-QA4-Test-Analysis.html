1. Introduction

1.1 Purpose

This Test Analysis document defines comprehensive test conditions, scenarios, and coverage analysis for Loox, a Shopify application designed for customer review collection and management. The analysis builds upon the Test Plan foundation to provide detailed, executable test conditions based on live system exploration and risk assessment.

1.2 System Under Test Overview

Loox is a Shopify app that enables merchants to collect, manage, and display customer reviews with photos and videos. The system operates as an embedded iframe-based application within the Shopify ecosystem, providing review collection, management, display widgets, email automation, and discount management capabilities.

Live System Observations:
  • Setup Status: Complete (Core script and widgets configured)
  • Current Data State: 1 review request sent, 0 reviews collected, 0 photo/video reviews
  • Performance Metrics: $0 reviews-generated revenue, $3.4K referrals-generated revenue
  • Main Modules: Reviews (active), Referrals (disabled), Upsell (disabled), Settings (active)
  • Email Configuration: Replies address set to slavastinov@gmail.com
  • Compliance Settings: Configured for marketing consent and U.S. customers

1.3 Test Analysis Objectives

The primary objectives of this test analysis are:

  1. Identify comprehensive test conditions for high-risk features
  2. Define specific, actionable test scenarios based on live system exploration
  3. Establish risk-based prioritization framework
  4. Ensure complete traceability to Test Plan risks and requirements
  5. Provide executable test conditions for automation and manual testing
  6. Validate system behavior under various conditions and edge cases

4. Use Case Analysis

4.1 Primary Use Cases

Based on live system exploration, the following primary use cases have been identified:

4.1.1 Review Collection Workflow

Use Case: Automated Review Request

Critical

Primary Actor: System (Automated)

Preconditions: Order completed, customer email available

Main Flow:

  1. System identifies completed order (e.g., "The Archived Snowboard" for aaa@abv.bg)
  2. System sends automated review request email
  3. Customer receives email with review link
  4. Customer submits review with optional photo/video
  5. System processes and stores review

Postconditions: Review collected and available for moderation

4.1.2 Email Configuration Management

Use Case: Email Settings Configuration

Critical

Primary Actor: Store Administrator

Preconditions: Administrator logged into Loox settings

Main Flow:

  1. Navigate to Settings > General > Email settings
  2. Configure email replies address (e.g., slavastinov@gmail.com)
  3. Set email compliance settings (marketing consent, U.S. customers)
  4. Configure unsubscribe settings
  5. Save email configuration

Postconditions: Email settings updated and active

4.1.3 Order Management and Search

Use Case: Order Search and Filtering

High

Primary Actor: Store Administrator

Preconditions: Orders exist in system

Main Flow:

  1. Navigate to Settings > Orders
  2. Enter search criteria (e.g., "aaa@abv.bg")
  3. Select filter options (All orders, Sent, Scheduled, etc.)
  4. Select time range (All time, Custom)
  5. Execute search
  6. Review search results

Postconditions: Filtered order list displayed

4.2 Secondary Use Cases

4.2.1 Branding Customization

Use Case: Logo and Style Customization

Medium

Primary Actor: Store Administrator

Main Flow:

  1. Navigate to Settings > Branding
  2. Upload logo (JPG, PNG, WEBP, JPEG, GIF up to 15MB)
  3. Customize star rating color (e.g., #EBBF20)
  4. Select corner style (Sharp, Slightly rounded, Rounded, Extra rounded)
  5. Choose main font (Default: Poppins)
  6. Configure hide Loox branding option
