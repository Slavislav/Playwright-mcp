1. Introduction

1.1 Purpose

This Test Design document defines comprehensive test cases and scenarios for Loox, a Shopify application designed for customer review collection and management. The design builds upon the Test Plan and Test Analysis foundations to provide detailed, executable test cases with appropriate test design techniques and automation opportunities.

1.2 System Under Test Overview

Loox is a Shopify app that enables merchants to collect, manage, and display customer reviews with photos and videos. The system operates as an embedded iframe-based application within the Shopify ecosystem, providing review collection, management, display widgets, email automation, and discount management capabilities.

Live System Observations:
  • Setup Status: Complete (Core script and widgets configured)
  • Current Data State: 1 review request sent, 0 reviews collected, 0 photo/video reviews
  • Performance Metrics: $0 reviews-generated revenue, $3.4K referrals-generated revenue
  • Main Modules: Reviews (active), Referrals (disabled), Upsell (disabled), Settings (active)
  • Email Configuration: Replies address set to slavastinov@gmail.com
  • Compliance Settings: Configured for marketing consent and U.S. customers

1.3 Test Design Objectives

The primary objectives of this test design are:

  1. Create detailed, executable test cases for high-risk features
  2. Apply appropriate test design techniques for comprehensive coverage
  3. Design test cases for automation opportunities
  4. Ensure complete traceability to Test Plan risks and Test Analysis conditions
  5. Provide specific test data sets and validation criteria
  6. Optimize test case design for competition scoring (20/20 points)

2. Test Design Techniques

2.1 Applied Test Design Techniques

Based on the Test Plan risks and Test Analysis conditions, the following test design techniques have been applied to create comprehensive test coverage:

2.1.1 State Transition Testing

Application: Email automation workflow, review status management, order processing states
Purpose: Test application state changes and transitions
Benefits: Visualizes state flows, identifies transition errors

2.1.2 Boundary Value Analysis

Application: File upload validation (15MB limit), email format validation, form field limits
Purpose: Test values at the edges of valid ranges
Benefits: Identifies edge case failures, validates boundary logic

2.1.3 Equivalence Partitioning

Application: Email format validation, file type validation, user role testing
Purpose: Test representative values from each equivalence class
Benefits: Reduces test cases while maintaining coverage

2.1.4 Pairwise Testing

Application: Email compliance settings, branding customization options
Purpose: Test combinations of input parameters efficiently
Benefits: Optimizes test case count while covering interactions

2.1.5 Error Guessing

Application: Edge cases, unusual user behaviors, system limitations
Purpose: Use experience to identify potential failure areas
Benefits: Identifies scenarios formal techniques might miss

3. Test Case Design

3.1 Critical Priority Test Cases (P1)

Based on Test Plan risk R-001 (Email delivery system failure) and Test Analysis conditions, the following critical test cases have been designed:

TC-EMAIL-001: Email Replies Address Configuration

Critical

Objective: Verify email replies address configuration functionality

Preconditions: Access to email settings, valid authentication

Test Steps:

  1. Navigate to Settings > General > Email settings
  2. Verify current email replies address displays "slavastinov@gmail.com"
  3. Clear the email address field
  4. Enter valid email format: "test@example.com"
  5. Save configuration
  6. Verify email address is updated
  7. Enter invalid email format: "invalid-email"
  8. Attempt to save

Expected Results: Valid email accepted and saved, invalid email rejected with error message

Automation Tool: Playwright - Automated functional testing
Priority: P1 - Must be automated

TC-EMAIL-002: Email Compliance Settings Validation

Critical

Objective: Verify email compliance settings functionality

Test Steps:

  1. Navigate to Email compliance section
  2. Verify "Unsubscribe customers from Loox and Shopify marketing emails" is selected
  3. Test "Keep customers subscribed to Shopify marketing emails" option
  4. Verify "Customers who gave marketing consent and all U.S. customers" is selected
  5. Test "Only customers who gave marketing consent" option
  6. Test "All customers" option
  7. Verify "Send transactional emails to customers who did not give marketing consent" is checked
  8. Save settings and verify persistence

Expected Results: All compliance options function correctly, settings save properly

Automation Tool: Playwright - Automated functional testing
Priority: P1 - Must be automated

TC-EMAIL-003: Email Template Customization

Critical

Objective: Verify email template customization functionality

Test Steps:

  1. Navigate to Email customization section
  2. Click "Edit" button for email appearance
  3. Test color customization options
  4. Test font selection options
  5. Test logo upload functionality
  6. Save customization settings
  7. Verify changes are applied

Expected Results: Email template customization works correctly, changes are applied

Automation Tool: Playwright - Automated functional testing
Priority: P1 - Must be automated

3.2 High Priority Test Cases (P2)

Based on Test Plan risks R-002, R-004, R-006, R-007, R-008 and Test Analysis conditions, the following high priority test cases have been designed:

TC-ORDER-001: Order Search Functionality

High

Objective: Verify order search functionality

Test Steps:

  1. Navigate to Settings > Orders
  2. Enter "aaa@abv.bg" in search box
  3. Click "Search" button
  4. Verify search results display order for "The Archived Snowboard"
  5. Verify order shows "Sent: 33 minutes ago" status
  6. Test search with invalid email format
  7. Test empty search

Expected Results: Valid search returns results, invalid search shows appropriate message

Automation Tool: Playwright - Automated functional testing
Priority: P2 - Must be automated

TC-ORDER-002: Order Filtering Options

High

Objective: Verify order filtering functionality

Test Steps:

  1. Navigate to Settings > Orders
  2. Test "All orders" filter option
  3. Test "Sent" filter option
  4. Test "Scheduled/Pending fulfillment" filter option
  5. Test "Review received" filter option
  6. Test "Blocked from Loox emails" filter option
  7. Test "Cancelled" filter option
  8. Test "All time" vs "Custom" time range options

Expected Results: All filter options work correctly, results update appropriately

Automation Tool: Playwright - Automated functional testing
Priority: P2 - Must be automated

TC-BRAND-001: Logo Upload Functionality

High

Objective: Verify logo upload functionality and validation

Test Steps:

  1. Navigate to Settings > Branding
  2. Click "Choose a file" button
  3. Upload valid JPG file (under 15MB)
  4. Upload valid PNG file (under 15MB)
  5. Upload valid WEBP file (under 15MB)
  6. Upload valid JPEG file (under 15MB)
  7. Upload valid GIF file (under 15MB)
  8. Test invalid file type (e.g., .txt)
  9. Test oversized file (>15MB)

Expected Results: Valid files accepted, invalid files rejected with appropriate error messages

Automation Tool: Playwright - Automated functional testing
Priority: P2 - Must be automated

  • @domain.com (missing username)

  • user..name@domain.com (double dots)

  • .pdf files (document files)
  • .doc files (Word documents)

  • #GGGGGG (invalid hex characters)
  • #12345 (too short)
